因为审批不知道什么时候才可以通过就直接拿了309一台空的机子试验了一下，还算顺利，已经docker部署成功了
感觉如果这样子部署docker，每次更新后全部重新来过有点大，所以未来如果有2.0的话，会想办法实现分离？大概 
docker build -t unmooredz/unmooredz-blog:v1.2 .  //构建Docker镜像
docker push unmooredz/unmooredz-blog:v1.2 //推送Docker镜像
docker pull unmooredz/unmooredz-blog:v1.2 //下载Docker镜像
docker stop ID //停下原来的docker
docker rm ID //删除原来的docker
docker run -d --name blog --restart unless-stopped -p 80:80 unmooredz/unmooredz-blog:v1.2 //运行Docker镜像保持开机启动
